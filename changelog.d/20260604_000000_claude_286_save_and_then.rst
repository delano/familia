Added
-----

- ``Horreum#save_and_then``: a post-save convenience method that yields
  ``self`` to a block only when ``save`` returns truthy. Unlike Ruby's
  ``.then`` (which always yields regardless of the receiver's truthiness),
  ``save_and_then`` skips the block on failure, so callers don't need an
  inner guard. The block's return value becomes the method's return value,
  enabling patterns like ``key = user.save_and_then { |u| u.dbkey }``. #286

  .. code-block:: ruby

     user.name = "Alice"
     user.save_and_then do |u|
       AuditLog.record(:user_updated, u.identifier)
     end

Documentation
-------------

- ``Horreum.create!``: added ``@yield``, ``@yieldparam``, and
  ``@yieldreturn`` YARD tags documenting the post-success block semantics
  that were present in the implementation but invisible in the API docs.
  The block receives the persisted instance and is skipped entirely when
  ``save_if_not_exists!`` raises (e.g. ``RecordExistsError``). #286
